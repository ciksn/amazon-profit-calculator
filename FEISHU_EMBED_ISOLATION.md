# 飞书网页卡片独立实例说明

当前飞书接入方式是普通网页卡片预览。该模式不会可靠地向网页提供当前云文档 ID，因此不能在复制模板后自动按文档 ID 建立数据绑定。

## 当前方案

1. 在选品模板中放置不带任何参数的 `embed.html` 通用链接。
2. 第一次打开时，服务端创建一个全新的测算实例。
3. 页面显示“已创建一个全新的独立测算”，点击“复制专属卡片链接”。
4. 用专属链接替换当前飞书文档中的通用卡片链接。
5. 后续刷新、协作和单站卡片都通过专属访问码进入同一个实例。

专属访问码位于 URL 的 `#key=...` 片段中，不会作为普通 HTTP 请求路径写入服务器访问日志。拥有专属链接的人可以访问该测算，因此它应当跟随飞书文档权限一起管理，不应发布到公开渠道。

## 安全边界

- 嵌入接口不会返回数据库品类列表。
- 缺少访问码、访问码错误时直接拒绝，不会回退到数据库第一条数据。
- 后端从访问码确定项目，忽略前端提交的其他项目 ID。
- 竞品和方案记录会校验是否属于当前实例。
- 嵌入卡片不能删除整个实例。

## 后续升级

改成飞书云文档小组件后，可以使用飞书提供的文档/组件上下文完成自动绑定，并取消“复制专属链接”这一步。现有 `/api/embed/*` 接口可以继续复用，只需把访问码换成由飞书身份和文档上下文签发的短期会话。
